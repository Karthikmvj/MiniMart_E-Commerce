========================================================================
                      MINIMART PROJECT EXPLANATION
========================================================================
Welcome to the developer explanation guide for MiniMart! This guide is 
written in very simple English for beginner developers. You can save 
this text file or print it to a PDF for your notes.

------------------------------------------------------------------------
1. WHAT IS THIS PROJECT?
------------------------------------------------------------------------
MiniMart is a simple online shopping website (an e-commerce platform).
It is like a mini-version of websites like Amazon or Flipkart.
* Regular users can visit the website, browse products, add them to a 
  shopping cart, and buy them.
* Store owners (admins) have a special panel where they can add, edit, 
  and delete products, view orders, and manage categories.

------------------------------------------------------------------------
2. WHY WAS THIS PROJECT CREATED & WHAT PROBLEM DOES IT SOLVE?
------------------------------------------------------------------------
* Why was it created?
  It was created to give customers a clean, fast, and simple way to shop
  online locally. It also provides a ready-made framework for shop owners
  to create an online presence quickly.
* What problem does it solve?
  1. For buyers: It makes shopping easy by letting them browse products,
     filter by category/price, save items in a wishlist, and checkout.
  2. For sellers: It solves the problem of tracking inventory, pricing, 
     and customer orders without needing expensive custom software.
  3. Multilingual support: It translates the store names, descriptions, 
     and buttons into Tamil (தமிழ்) so that local buyers who do not speak 
     English can easily use it.

------------------------------------------------------------------------
3. WHY WAS THIS PROJECT NAME CHOSEN?
------------------------------------------------------------------------
The name "MiniMart" was chosen because:
* "Mini" means small and lightweight. The code is clean and fits small 
  to medium business needs.
* "Mart" means a market or shop. 
Together, it is a lightweight digital market built for everyday shopping.

------------------------------------------------------------------------
4. WHICH TECHNOLOGIES ARE USED?
------------------------------------------------------------------------
This project is built using a modern, clean web technology stack:
* Backend Framework: Django (Python). Django is a very secure and fast 
  framework for building websites. It handles our database, URL routing, 
  and templates.
* Database: SQLite3. A lightweight file-based database that stores all 
  the products, users, cart details, and orders.
* Frontend: HTML5, CSS3, and JavaScript. 
  - Bootstrap CSS is used to make the pages look clean and work well on 
    mobile phones.
  - Custom JavaScript is used to handle image zoom, carousel switching, 
    and UI animations.
* Translation System: Django Translation (i18n). This helps us run the 
  website in multiple languages (English and Tamil).

------------------------------------------------------------------------
5. WHAT ARE THE MAIN FEATURES?
------------------------------------------------------------------------
1. Dual Language Mode: Users can switch the website language between 
   English and Tamil.
2. Product Catalog: Products are grouped by category (Electronics, 
   Groceries, Fashion, etc.) with filter and search functions.
3. Multiple Images with Zoom: Each product page has a main photo and 
   several detail photos. You can hover your mouse over photos to zoom 
   in and see the texture/details.
4. Wishlist: Registered users can save products they like for later.
5. Interactive Shopping Cart: Add items, change quantities, and check out.
6. Orders Tracking: Users can enter their order numbers to track shipping status.
7. Admin Dashboard: A separate panel for admins to manage the entire shop.

------------------------------------------------------------------------
6. EXPLAIN EACH PAGE IN SIMPLE WORDS
------------------------------------------------------------------------
* Home Page (home.html): 
  The front page of the shop. It shows a list of featured categories and 
  recent products. It also has banners and promotions.
* Product List Page (product_list.html): 
  Displays all products. It has a sidebar where users can filter products 
  by category, brand, and sort them by price (low to high / high to low).
* Product Detail Page (product_detail.html): 
  Shows a single product. It has zoomable images, description, stock status, 
  original and discount prices, and user reviews.
* Shopping Cart Page (cart.html): 
  Lists all products a user has added to buy. It calculates the total cost.
* Checkout Page (checkout.html): 
  The page where the user enters their delivery name, phone, and address.
* Payment Options Page (payment.html): 
  Where the user selects cash on delivery or enters card/UPI details.
* Order Success Page (order_success.html): 
  Shows a thank you message, the generated Order Number, and tracking links.
* Admin Signup/Login Pages: 
  Special forms for store managers to access the dashboard securely.
* Admin Panel Pages: 
  Screens to add new categories, manage stock, view list of orders, and update products.

------------------------------------------------------------------------
7. EXPLAIN THE FOLDER STRUCTURE IN SIMPLE WORDS
------------------------------------------------------------------------
Here is the folder structure simplified:
- MiniMart/                  <- The main folder of our project
  - MiniMart/                <- The configuration folder for Django
    - settings.py            <- Settings for Database, Languages, Apps
    - urls.py                <- The master list of links (routing)
  - store/                   <- The "Store App" (handles products, cart, orders)
    - models.py              <- Defines database tables (Product, Category, Order)
    - views.py               <- Contains backend logic (how pages load, add to cart)
    - urls.py                <- Defines urls/links for the store
    - translations.py        <- Stores Tamil words for name/description translations
  - accounts/                <- The "Accounts App" (handles customer signup, login, payment methods)
  - adminpanel/              <- The "Admin App" (handles dashboard and management screens)
  - templates/               <- Contains all HTML pages
    - store/                 <- HTML templates for product list, detail, cart
    - accounts/              <- HTML templates for login, profile
  - media/                   <- Stores product images uploaded/downloaded
    - photos/products/       <- Folder containing downloaded images from Unsplash
  - db.sqlite3               <- Our SQLite database file
  - manage.py                <- The Django command runner utility

------------------------------------------------------------------------
8. EXPLAIN USER FLOW (START TO FINISH)
------------------------------------------------------------------------
Here is how a buyer uses the site:
1. Signup/Login: The user visits the site and logs in.
2. Browse: They browse products on the Home Page or Product List.
3. Select: They click "View Details" to see the zoomable images and description.
4. Cart: They click "Add to Cart". They can increase or decrease quantities.
5. Checkout: They click "Checkout" and fill in their shipping address.
6. Pay: They select payment option and click "Confirm Payment".
7. Order: They receive their order number and can track its status.

------------------------------------------------------------------------
9. EXPLAIN ADMIN FLOW (START TO FINISH)
------------------------------------------------------------------------
Here is how a store manager runs the shop:
1. Access: The admin signs in to the special Admin Panel.
2. Categories: They click "Categories" to create groups like "Groceries".
3. Products: They click "Add Product", fill details, set pricing, upload 
   photos, and save.
4. Orders: They view orders placed by customers, check shipping addresses, 
   and mark them as "Pending", "Shipped", or "Delivered".

12. COMMON TEAM LEAD QUESTIONS & SIMPLE ANSWERS
------------------------------------------------------------------------
Q: Where is the product seeding data located?
A: In add_ten_products.py (in the root directory) and store/seed_data.py.

Q: How do we synchronize/refresh images?
A: We run the "add_ten_products.py" script or "update_images.py" script, 
   which automatically pulls matching Unsplash photo assets to the media folder.

Q: How is the language translation handled?
A: We use Django's built-in i18n middleware. In store/models.py, we have 
   translated properties that fetch Tamil strings from store/translations.py 
   if the URL prefix is set to /ta/.

Q: What happens if a seeding download fails?
A: The script will output a warning log, and the database will safely 
   retain its current product image without breaking.
